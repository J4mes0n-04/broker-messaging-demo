# Архитектура системы 

## Компоненты
1. producer — сервис, отправляющий заказы.
2. consumer — сервис, обрабатывающий заказы.
3. Kafka — хранение и маршрутизация сообщений.
4. Schema Registry — управление схемами Avro.

## Потоки данных
1. Продюсер → Kafka (топик `orders`) → Консьюмер.
2. Схема сообщения регистрируется в Schema Registry.

## Стратегии
- Партиции: по `order_id` (равномерное распределение).
- Репликация: фактор 1 (для демо).
- Срок хранения: 7 дней.
