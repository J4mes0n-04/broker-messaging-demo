# Требования к системе обмена сообщениями

## Функциональные требования
1. **Отправка сообщений**:
   - Продюсер должен публиковать сообщения в топик `orders`.
   - Сообщения должны содержать: `order_id`, `customer_id`, `amount`, `timestamp`.

2. **Приём сообщений**:
   - Консьюмер должен читать сообщения из топика `orders`.
   - Должна быть поддержка групп потребителей.

3. **Гарантия доставки**:
   - Не менее 1 раза (at-least-once).
   - Обработка дубликатов.

## Нефункциональные требования
- Задержка доставки: ≤ 100 мс.
- Пропускная способность: ≥ 1000 сообщений/сек.
- Доступность: 99.9%.

## Ограничения
- Формат сообщений: JSON.
- Сериализация: Avro (через Schema Registry).
- Максимальный размер сообщения: 1 МБ.
